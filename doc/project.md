# Shadow_R 项目文档

## 1. 项目概述

Shadow_R 是一个基于深度学习的阴影去除项目，采用两阶段训练策略，包含阴影去除模块和精细化模块。

对于数据集，已经准备好了包含阴影和无阴影图像的数据集，数据集结构如下：

   └── val/

## 2. 数据增强 (Data Augmentation)

为了提高模型的泛化能力，采用了以下增强手段：

- **随机裁剪 (Random Cropping)**：裁剪为 384 × 384 像素的图块（patches）
- **随机旋转 (Random Rotations)**：随机旋转 90、180 或 270 度
- **翻转 (Flipping)**：包括垂直翻转和水平翻转

## 3. 优化器配置 (Optimizer)

- **类型**: Adam 优化器
- **超参数**: 
  - β₁ = 0.9
  - β₂ = 0.999

## 4. 两阶段训练策略

模型采取分阶段训练的方式：

### 第一阶段 (Stage I)

- **优化目标**：仅优化阴影去除模块 (Shadow Removal module)
- **学习率 (Learning Rate)**：初始设定为 1×10⁻⁴，随后逐渐降低至 6.25×10⁻⁶
- **损失函数**：
  ```
  L_loss = L₁ + α·L_SSIM + β·L_Percep + γ·L_adv
  ```

### 第二阶段 (Stage II)

- **优化目标**：同时更新阴影去除模块和精细化模块 (Refinement module) --联合训练
- **学习率**：采用 1×10⁻⁵ 的恒定学习率
- **损失函数**：去除了对抗损失 (L_adv)，仅保留基础重建损失、结构相似性损失和感知损失
  ```
  L_loss = L₁ + α·L_SSIM + β·L_Percep
  ```

## 5. 损失函数权重系数 (Loss Hyper-parameters)

在优化过程中，各项损失的权重配置如下：

| 损失类型 | 权重符号 | 权重值 | 说明 |
|---------|---------|--------|------|
| MS-SSIM 损失 | α | 0.2 | 结构相似性损失 |
| 感知损失 | β | 0.01 | 基于VGG的感知损失 |
| 对抗损失 | γ | 0.0005 | 对抗训练损失 |

## 6. 损失函数详解

### 6.1 基础重建损失 (L₁)

- **类型**: L1 损失
- **作用**: 计算预测图像与目标图像的像素级差异
- **特点**: 对异常值敏感，适合图像重建任务

### 6.2 结构相似性损失 (L_SSIM)

- **类型**: MS-SSIM (多尺度结构相似性)
- **作用**: 保持图像的结构信息和纹理细节
- **权重**: α = 0.2

### 6.3 感知损失 (L_Percep)

- **类型**: 基于VGG19的感知损失
- **作用**: 提取高层语义特征，确保生成图像在语义层面的一致性
- **实现**: 使用预训练的VGG19网络提取特征
- **权重**: β = 0.01

### 6.4 对抗损失 (L_adv)

- **类型**: 非饱和对抗损失 + R1梯度惩罚
- **作用**: 通过判别器提升生成图像的真实感
- **实现**: 
  - 使用softplus损失函数
  - 添加R1正则化防止模式崩溃
- **权重**: γ = 0.0005
- **使用阶段**: 仅在第一阶段使用

## 9. 评估指标

### 9.1 客观指标

- **PSNR (峰值信噪比)**: 衡量图像质量的客观指标

## 10. 项目技术要求

### 10.1 训练框架

- **框架**: 必须使用 **PyTorch Lightning (PL)** 框架进行训练
- **优势**: 
  - 简化训练循环和分布式训练配置
  - 自动处理设备和混合精度
  - 内置日志和检查点管理

### 10.2 分布式训练

- **要求**: 支持多卡（多 GPU）训练
- **实现方式**: 使用 PyTorch Lightning 的 DDP 策略
- **信息打印**: 
  - 所有日志信息仅在**主进程 (rank=0)** 上打印
  - 避免多进程重复输出造成混乱
  - 使用 `self.global_rank == 0` 或 `trainer.is_global_zero` 进行判断

### 10.3 实验记录 (SwanLab)

- **工具**: 使用 **SwanLab** 进行实验管理和可视化
- **记录内容**:
  - **训练损失 (Loss)**: 每个 step/epoch 的训练损失
  - **验证指标 (Val Metrics)**: 
    - PSNR (峰值信噪比)
    - SSIM (结构相似性)
    - LPIPS (学习感知图像块相似性)
  - **学习率 (LR)**: 每个 epoch 的学习率变化

### 10.4 检查点保存 (Checkpoint)

- **要求**: 必须保存模型检查点
- **保存策略**:
  - **最新模型 (Latest)**: 每个 epoch 结束后保存，便于断点续训
  - **最佳模型 (Best)**: 根据验证集指标（如 PSNR）保存性能最好的模型
- **保存内容**:
  - 模型权重 (model state_dict)
  - 优化器状态 (optimizer state)
  - 学习率调度器状态 (scheduler state)
  - 当前 epoch 和全局 step
  - 最佳验证指标记录
- **命名规范**: `best_model.pth`, `latest_model.pth`

## 11. 额外说明
1. 默认从头训练，如需断点续训，需要设置 `model.resume` 参数，指定检查点路径。
2.采用cosine学习率调度器时，第一阶段训练结束后，需手动调整初始学习率为1e-5，以适应第二阶段的恒定学习率需求。
3.只保存最佳模型和最新模型，避免存储空间浪费。
